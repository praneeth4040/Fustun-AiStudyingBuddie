# Contributing

Thanks for your interest in contributing to Fustun AI Assistant! We welcome contributions of all kinds â€” issues, bug fixes, documentation improvements, and new features.

How to contribute
1. Fork the repository and create a feature branch.
2. Run the extension locally (see README) and verify changes.
3. Open a pull request with a clear description of the change and rationale.

Guidelines
- Keep changes small and focused.
- Write clear commit messages and PR descriptions.
- Include tests or manual verification steps for new functionality where applicable.

Reporting issues
- Use the issue templates when opening bug reports or feature requests.

Code style
- JavaScript files follow standard ES module syntax; keep code consistent with the existing project style.

Security
- Do not include secrets (API keys, passwords) in commits. If you need to share a secret for reproduction, use a secure channel.
